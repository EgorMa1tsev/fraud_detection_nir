# Применение сети Хопфилда  для идентификации мошеннических банковских транзакций

В этой работе реализована сеть Хопфилда для идентификации мошеннических банковских транзакций

В мире регулярно происходят мошеннические транзакции, и хотя основным методом их идентификации являются классификаторы (такие как решающие деревья, random forest и их модификации), они часто неэффективны при работе с зашумленными или неполными данными, что требует применения иных нейросетевых архитектур.

В этой работе я рассмотрел альтернативный способ выявления мошеннических транзакций с применением сети Хопфилда.

Цель исследования: разработка сети Хопфилда для идентификации мошеннических транзакций.


Задачи:


1. Предобработка данных

2. Выбор паттернов среди легитимных и мошеннических транзакций

3. Обучение сети Хопфилда

4. Подбор гиперпараметров (длина бинарного вектора, количество паттернов, количество итераций)

5. Тестирование и сравнение метрик сети Хопфилда и решающего дерева

## Исходные данные

Данные были взяты на kaggle https://www.kaggle.com/datasets/computingvictor/transactions-fraud-datasets/data

Это комплексный набор финансовых данных из нескольких файлов, созданный Caixabank Tech для хакатона по ИИ 2024 года. Датасет объединяет записи транзакций, профили клиентов и данные по банковским картам за период 2010-х годов. Он предназначен для задач обнаружения мошенничества, анализа поведения клиентов и финансового прогнозирования.

transactions_data.csv	~1.42 ГБ (основной файл)	Детальные записи о транзакциях: суммы, дата/время, информация о merchant'ах. Идеально подходит для анализа паттернов трат и построения моделей обнаружения мошенничества. 12 стобцов 13 млн строк

cards_data.csv	~510 КБ	Информация по кредитным и дебетовым картам: лимиты, типы карт, даты активации. Связывается с клиентами через card_id. 13 столбцов 6145 строк

users_data.csv	(входит в общий объем)	Демографические данные клиентов и информация об их счетах. Позволяет проводить сегментацию пользователей и персонализированный анализ.

users_data.csv	(входит в общий объем)	Демографические данные клиентов и информация об их счетах. Позволяет проводить сегментацию пользователей и персонализированный анализ. 14 столбцов 2000 строк

train_fraud_labels.json	(входит в общий объем)	Бинарные метки (мошенническая/легитимная транзакция) для обучения моделей с учителем.
